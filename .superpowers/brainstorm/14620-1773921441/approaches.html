Three Implementation Approaches
How we structure the 4-phase build. Click the one you prefer.
A
Ship Fast, Add Backend Later
Phase 1 is 100% client-side. Backend added in Phase 2.
Phase 1 launches with zero backend — just Next.js + PNG cards + React state. The game works fully in the browser. Paywall gates are UI-only (honour system). In Phase 2, Supabase + Stripe are bolted on and gates become real.

Best if you want players using the game as fast as possible and are willing to add auth/payments in a second sprint.
✓ Fastest to launch ✓ Zero infra cost in Phase 1 ✓ No Supabase setup needed yet ✗ No real paywall enforcement until Phase 2 ✗ Some Phase 2 refactor needed
P1: Game + Landing
P2: Supabase + Stripe
P3: Multiplayer
P4: Polish + Growth
★ Recommended
B
Solid Foundation First
Supabase + auth set up in Phase 1. Stripe in Phase 2.
Phase 1 scaffolds the game AND sets up Supabase auth so the paywall gates are real from day one. Stripe subscriptions come in Phase 2 but the user system is already in place, making Phase 2 straightforward. No rework needed between phases.

Best if you want to start collecting emails / users immediately and avoid retrofitting auth later. The extra setup in Phase 1 is a few hours, not days.
✓ Real user accounts from launch ✓ No auth rework in Phase 2 ✓ Can gate features properly from day 1 ~ 1–2 extra days of Phase 1 setup ✗ Supabase project needs configuring upfront
P1: Game + Auth + Landing
P2: Stripe + Paywall
P3: Multiplayer
P4: Polish + Growth
C
Full Stack from Day One
Supabase + Stripe + auth all in Phase 1.
Everything — game engine, auth, Stripe subscriptions, and paywall gates — is wired up before going live. Phase 1 is a complete, monetisable product. Later phases add multiplayer and polish only.

Best if you want to launch once and have it immediately ready to take payment. Higher Phase 1 complexity, but you're production-ready from the first deploy.
✓ Monetisable from first launch ✓ Single big build, then polish ✗ Longest Phase 1 ✗ More moving parts before you can play-test ~ 3–4 weeks before first deploy
P1: Game + Auth + Stripe + Paywall
P2: Multiplayer
P3: Dashboard + Cards
P4: Polish + Growth
