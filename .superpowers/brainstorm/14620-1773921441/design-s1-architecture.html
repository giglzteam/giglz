Card Data Model (Simplified)

Because die→type mapping is consistent across all cards, we don't need card text in the database. The app stores only the card image filenames. The PNG is the card.

// lib/game/cards.ts // All 75 card image paths (public/cards/26.png → 100.png) export const CARD_IDS = [26, 27, ..., 100] // 75 cards // Fixed die → challenge type mapping (same on every card) export const DIE_MAP = { 1: { type: 'words', label: 'Words', desc: 'Explain without saying the word', color: '#EA6CAE' }, 2: { type: 'cliche', label: 'Cliché', desc: 'Complete the well-known phrase', color: '#7ADDDA' }, 3: { type: 'associations', label: 'Associations', desc: 'Say words that relate to the topic', color: '#68519E' }, 4: { type: 'gestures', label: 'Gestures', desc: 'Act it out — no talking!', color: '#3097D1' }, 5: { type: 'persona', label: 'Persona', desc: 'Act as the character or celebrity', color: '#7ADDDA' }, 6: { type: 'dares', label: 'Dares', desc: 'Complete the dare!', color: '#EA6CAE' }, } // Game state (pure client-side, no DB in Phase 1) interface GameState { players: { name: string; emoji: string; score: number }[] currentPlayer: number deck: number[] // shuffled card IDs deckIndex: number // current position in deck dieValue: number | null phase: 'setup' | 'rolling' | 'reveal' | 'finished' cardsToWin: number // default 10 }

4-Phase Build Plan (Approach B)

Phase 1
Core Game + Auth + Landing
Next.js scaffold · brand tokens · PNG card display · game engine · player setup · die roll · /play page · Supabase auth (email + Google) · basic paywall gates (UI) · landing page · PWA · deploy to Vercel
Phase 2
Subscriptions + Real Paywall
Stripe products (Plus €2.99 · Pro €5.99) · checkout route · webhook handler · pricing page · paywall gates enforced server-side · dashboard (plan info + billing)
Phase 3
Multiplayer Rooms (Pro)
Supabase Realtime · room creation · 6-char join code + QR · guest join (no auth needed) · host controls game · all guests see same card · reconnect handler
Phase 4
Polish + Growth
Custom card editor · game history · seasonal packs · share results screen · SEO · analytics · email drip to upsell
